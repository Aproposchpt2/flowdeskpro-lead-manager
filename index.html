Phase 1 Foundation

Built for repeatable client deployment.

The package is designed to be cloned, branded, connected to a client Supabase project, and deployed to Netlify without placing backend secrets in the browser.

Intake

Lead capture form

Collects contact details, service need, urgency, callback preference, and customer message through a polished public form.

Dashboard

CRM queue and detail panel

Loads records through Netlify Functions and gives operators filters, metrics, status control, notes, next action, and email follow-up.

Backend

Server-side REST architecture

Netlify Functions call Supabase REST and Resend using environment variables. Frontend files never contain service keys.
